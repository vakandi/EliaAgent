# Business Tools Skill

## Description
Build your business infrastructure for 2026 and beyond with Alex Hormozi's frameworks. This skill covers AI adoption, workflow-based thinking, the agent revolution, and why "boring businesses make lots of money" through systematic execution.

## Core Principles

### 1. Workflow Over Role-Based Thinking
> "2026 - shift away from role-based thinking to workflow-based thinking."

Stop thinking "I need a marketer" and start thinking "I need this workflow automated."

### 2. Agents Are Eating Workloads
> "Agents are eating up more and more workload so one person can now do the work of three, then five, then 10."

The leverage is unprecedented. One person with good tools beats a team with bad ones.

### 3. The Idea-to-Output Pipeline
> "The first thing would be an idea → trend research → ranking → scripting → final output."

AI makes the entire pipeline faster. Structure your workflow to maximize leverage.

### 4. AI Makes Your Job Easier
> "AI making it easier for customers to build their own = your job just got easier."

When customers can self-serve, your support load drops. When AI handles routine, your value increases.

### 5. Boring Businesses Win
> "Boring businesses make lots of money - repeat successful actions, do the boring work."

Excitement is overrated. Consistency compounds. The unsexy businesses are often the most profitable.

### 6. Content Quality First, Ads Second
> "Quality content first, then run best content as ads."

Build organic proof before paying to scale. Test with content, then amplify.

```

├── CRM (HubSpot, Pipedrive, or Notion)
├── Communication (Slack, Discord)

AI AGENT LAYER:

├── Writing Agent (Claude, GPT)
├── Design Agent (Midjourney, Figma AI)
├── Code Agent (Cursor, Copilot)
└── Data Agent (Zapier, Make)

CONTENT PIPELINE:
├── Idea → Trend research (AI)
├── Ranking → Content calendar (AI)
├── Scripting → First drafts (AI)
├── Editing → Human refinement
└── Distribution → Automated posting

AUTOMATION SEQUENCE:
Trigger → AI handles → Human reviews → Published
                    ↓
            Exception handling
            (human only when needed)
```

### Framework 2: The Workflow Design Template

```
FOR ANY REPEATED TASK:

1. DOCUMENT THE CURRENT PROCESS
   Step 1: [What happens now]
   Step 2: [What happens now]
   Step 3: [What happens now]

2. IDENTIFY BOTTLENECKS
   - Which step takes longest?
   - Which step requires human judgment?
   - Which step is most error-prone?

3. DESIGN AI ENHANCEMENTS
   - Step 1: [AI can do X]
   - Step 2: [AI can do Y, human does Z]
   - Step 3: [Human only for approval]

4. IMPLEMENT AND TEST
   - Run 10 iterations
   - Track time saved
   - Refine the handoffs

5. SCALE OR ITERATE
   - Working well? → Apply to similar workflows
   - Not working? → Rethink the approach

EFFICIENCY TARGET:
If AI can do 70%+ of the work → Automate
If human required for 70%+ → Optimize the human work
```

## Scripts & Templates

### Script 1: The AI Tool Evaluation Checklist

```
BEFORE ADOPTING ANY AI TOOL, ASK:

1. WHAT PROBLEM DOES IT SOLVE?
   - Specific problem?
   - Current workaround?
   - Cost of not solving?

2. HOW DOES IT FIT THE WORKFLOW?
   - Part of existing workflow?
   - New workflow entirely?
   - Replace or enhance?

3. WHAT'S THE LEARNING CURVE?
   - Time to proficiency?
   - Team adoption needed?
   - Documentation available?

4. WHAT'S THE ROI?
   - Cost of tool?
   - Time saved?
   - Value of time saved?
   - Break-even timeline?

5. WHAT'S THE SWITCHING COST?
   - Data lock-in?
   - Integration dependencies?
   - Retraining needed?

DECISION: If ROI > 3x in 6 months, adopt. If not, optimize current tools first.
```

### Script 3: The "Should I Automate This?" Decision Tree

```
IS THIS TASK...
├── REPEATEDLY DONE (Same process, multiple times)
│   └── GO TO: Automate potential
├── ONE-TIME ONLY
│   └── DON'T AUTOMATE (ROI too low)
└── VARIABLE (Different each time)
    └── HUMAN ONLY (Automation too rigid)

AUTOMATE POTENTIAL:
├── TAKES > 10 MINUTES?
│   └── YES → Strong candidate
├── ERROR-SENSITIVE?
│   └── YES → Strong candidate (consistency)
├── NO CREATIVE INPUT NEEDED?
│   └── YES → Strong candidate
└── FREQUENTLY REPEATED?
    └── YES → Strong candidate

IF 3+ YES → AUTOMATE
IF 1-2 YES → EVALUATE CAREFULLY
IF 0 YES → DON'T AUTOMATE
```

## Anti-Patterns (What NOT to Do)

1. **Don't adopt tools without workflows** - Tools without process just automate chaos.

2. **Don't automate too early** - Document and optimize before automating inefficiencies.

3. **Don't replace humans with AI when judgment is needed** - Some decisions require context AI doesn't have.

4. **Don't forget the human element** - AI handles volume; humans handle nuance.

5. **Don't adopt every new tool** - More tools = more complexity = less execution.

6. **Don't ignore boring businesses** - They're boring because they work. Stop chasing shiny objects.

7. **Don't forget to measure ROI** - Every tool should pay for itself. If it doesn't, cut it.
